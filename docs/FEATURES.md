# Дополнительные возможности для Code Analysis

## Анализ возможностей

### 1. Поиск зависимостей между модулями
**Описание**: Найти все файлы, которые импортируют определенный модуль или класс.

**Потенциальная команда**: `find_dependencies`
- Параметры: `root_dir`, `module_name`, `dependency_type` (import/import_from)
- Возвращает: список файлов с номерами строк

**Польза для AI**: Понимание влияния изменений на другие части кодовой базы

### 2. Анализ сложности кода
**Описание**: Вычислить цикломатическую сложность методов и классов.

**Потенциальная команда**: `analyze_complexity`
- Параметры: `root_dir`, `file_path` (optional), `method_name` (optional)
- Возвращает: сложность для каждого метода/класса

**Польза для AI**: Определение методов, требующих рефакторинга

### 3. Поиск дубликатов кода
**Описание**: Найти дублирующиеся фрагменты кода.

**Потенциальная команда**: `find_duplicates`
- Параметры: `root_dir`, `min_lines` (минимальная длина дубликата)
- Возвращает: группы дублирующихся фрагментов

**Польза для AI**: Выявление возможностей для вынесения общего кода

### 4. Анализ покрытия тестами
**Описание**: Проверить какие методы покрыты тестами.

**Потенциальная команда**: `test_coverage`
- Параметры: `root_dir`, `test_dir`
- Возвращает: список непокрытых методов

**Польза для AI**: Понимание что нужно протестировать

### 5. Поиск антипаттернов
**Описание**: Найти известные антипаттерны в коде.

**Потенциальная команда**: `find_antipatterns`
- Параметры: `root_dir`, `pattern_type` (optional)
- Возвращает: список найденных антипаттернов

**Польза для AI**: Выявление проблемных мест в коде

### 6. Генерация документации
**Описание**: Автоматически генерировать документацию из докстрингов.

**Потенциальная команда**: `generate_docs`
- Параметры: `root_dir`, `output_format` (markdown/html)
- Возвращает: сгенерированную документацию

**Польза для AI**: Автоматическое создание документации

### 7. Анализ изменений
**Описание**: Сравнить два состояния кодовой базы.

**Потенциальная команда**: `analyze_changes`
- Параметры: `root_dir`, `base_commit`, `head_commit`
- Возвращает: список изменений с анализом

**Польза для AI**: Понимание что изменилось между версиями

### 8. Поиск неиспользуемого кода
**Описание**: Найти неиспользуемые методы, классы, импорты.

**Потенциальная команда**: `find_unused_code`
- Параметры: `root_dir`, `code_type` (method/class/import)
- Возвращает: список неиспользуемых элементов

**Польза для AI**: Определение что можно безопасно удалить

### 9. Анализ безопасности
**Описание**: Поиск потенциальных уязвимостей.

**Потенциальная команда**: `security_scan`
- Параметры: `root_dir`, `severity` (low/medium/high)
- Возвращает: список найденных проблем безопасности

**Польза для AI**: Выявление уязвимостей до продакшена

### 10. Метрики кода
**Описание**: Получить различные метрики кодовой базы.

**Потенциальная команда**: `get_metrics`
- Параметры: `root_dir`, `metric_type` (all/lines/complexity/dependencies)
- Возвращает: метрики кода

**Польза для AI**: Общая картина состояния кодовой базы

## Приоритеты реализации

1. **Высокий приоритет**:
   - Поиск зависимостей (критично для рефакторинга)
   - Анализ сложности (помогает принимать решения)
   - Поиск неиспользуемого кода (очистка кодовой базы)

2. **Средний приоритет**:
   - Поиск дубликатов
   - Анализ покрытия тестами
   - Метрики кода

3. **Низкий приоритет**:
   - Генерация документации
   - Анализ изменений
   - Поиск антипаттернов
   - Анализ безопасности
