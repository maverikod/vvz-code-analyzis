Каноничное правило:
Метрики валидного кода  
- соответствие правилам проекта
- отсутствие : 
- хардкода, 
- плейсхолдеров,
- обратной совместимости и фоллбеков, если пользователь или задание не оговаривают отдельно
- отсутствие недописанного кода
    - TODO
    - NotImplemented вне тела абстрактных методов
    - pass вне тела исключений
- отклонение от правил проекта и пользователя
- отклонения от поставленного задания
- абстрактные методы с кодом кроме NotImplemented, комментарием и докстрингов.
Метрики валидного задания 
- точное указание роли модели
- непротиворечивость внутренняя
- полнота
- точность
- непротиворечивость. правилам проекта
- КРИТИЧНО - 100% готовность для передачи данных модели.
- Обязательные блекстопы
- Обязательная повторная провера и исправления после написания кода
Метрики валидного плана:
- разбиение на шаги в отдельном каталоге
- 1 шаг = 1 файл кода = 1 файл описания шага
- Последовательность шагов с учетом зависимостей шагов
- каждый шаг предоставляет за счет ссылок контекст необходимый для выполения задачи и является самодостаточным
- Отдельный файл с описанием цепочек для параллельного задания выполнения шагов
