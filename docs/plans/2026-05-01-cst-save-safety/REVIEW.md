## Найденные проблемы

### A. Критические

#### A1. Пропущено обновление снимка после save

**Где:** шаги 04, 05, 06 — ни один не описывает обновление `disk_source_sha256_hex` + `disk_source_length` на дереве **после** успешной записи.

**Проблема:** после `cst_save_tree` или save из `cst_modify_tree` файл на диске изменился, но снимок на `CSTTree` хранит хэш **старого** файла. Повторный save без reload → `FILE_CHANGED_SINCE_LOAD`, хотя файл менял сам сервер.

**Решение:** в конце `save_tree_to_file`, после успешного readback verify, вызвать `_attach_disk_snapshot(tree, source_code)` (из шага 02) или присвоить поля напрямую. Добавить в **шаг 04** явный пункт.

#### A2. Replay: отсутствует `finally`-блок для `remove_tree`

**Где:** шаг 03, раздел 4.

**Проблема:** если `modify_tree(new_tree_id, ...)` бросает исключение, `remove_tree(new_id)` не вызывается → утечка временного дерева в `_trees`.

**Решение:** обернуть replay в `try/finally` с `remove_tree` в `finally`. Добавить явную инструкцию в шаг 03.

#### A3. `assert_replay_matches` читает диск повторно

**Где:** шаг 03, API `assert_replay_matches`.

**Проблема:** функция принимает `target_path` и читает файл с диска, хотя в шаге 04 replay вызывается после disk snapshot check → текст с диска уже верифицирован и доступен. Лишнее чтение I/O в критическом участке под lock.

**Решение:** изменить сигнатуру: принимать `original_source: str` вместо `target_path`. В шаг 04 передавать уже прочитанный текст.

### B. Неточности в ссылках

#### B1. Шаг 05: строки 214–266 / 248–255

**Факт:** `CSTModifyTreeCommand.execute` на строках 62–312 (file_structure: start_line=62, end_line=312). Указанные в плане диапазоны не соответствуют текущему коду.

**Решение:** убрать конкретные номера строк; использовать grep-указатели (`grep 'save_tree_to_file' в execute`).

#### B2. Шаг 06: metadata на ~454+

**Факт:** метод `metadata` на строках 315–322 (file_structure: start_line=315, end_line=322). Класс `CSTModifyTreeCommand` заканчивается на 322.

**Примечание:** шаг 06 указывает на `cst_save_tree_command.py`, а не `cst_modify_tree_command.py`. Ссылка `~454+` неверна — проверить реальное расположение `metadata` в `cst_save_tree_command.py`.

### C. Неопределённости для модели-исполнителя

#### C1. Стиль ошибок не зафиксирован

**Где:** шаг 06, раздел 1 — «согласовать: либо исключения, либо dict».

**Решение:** зафиксировать: **исключения** (`SaveVerificationError`). Единообразно с тем, что шаг 03 уже определяет класс исключения, а шаг 04 его ловит.

#### C2. `force_overwrite` — нет директивы

**Где:** README, раздел «Документация MCP».

**Решение:** явно указать: **не реализовывать в первой фазе**.

#### C3. Шаг 02: `encode` вызывается дважды

**Где:** раздел 3.1 — `hashlib.sha256(source.encode("utf-8")).hexdigest()` и `len(source.encode("utf-8"))`.

**Решение:** добавить инструкцию кэшировать `source_bytes = source.encode("utf-8")` и использовать для обоих вычислений.

### D. Риски

#### D1. `save_tree_to_file` → рост размера

Сейчас 276 строк. С проверками (disk check, replay, readback, rollback) → ~340–370 строк. Лимит 400. Близко, но допустимо при аккуратной реализации.

#### D2. Replay в критическом участке под lock

CPU-intensive операция (parse + modify) на каждый save. На файлах 300+ строк — 50–200ms. Упомянут в PARALLELIZATION_MAP, но нет порога отключения.

#### D3. Тестирование replay изолированно

Replay зависит от `create_tree_from_code` + `modify_tree` + `remove_tree`. Для unit-тестов в шаге 07 потребуется либо мок, либо реальная инфраструктура `_trees`. Шаг 07 упоминает `MagicMock` для DB, но не для tree infrastructure.

## Необходимые изменения перед передачей в исполнение

1. **Шаг 04:** добавить пункт обновления снимка после save (A1).
2. **Шаг 03:** добавить `try/finally` для cleanup replay-дерева (A2); исправить сигнатуру `assert_replay_matches` (A3).
3. **Шаги 05, 06:** исправить ссылки на строки (B1, B2).
4. **Шаг 06:** зафиксировать стиль ошибок = исключения (C1).
5. **README:** зафиксировать `force_overwrite` = вне объёма (C2).
6. **Шаг 02:** кэширование `source_bytes` (C3).
7. **Шаг 07:** добавить тест «save → повторный save без reload» (A1).
