## 1. Цель шага

Расширить dataclass **`CSTTree`** полями **снимка диска** на момент последней синхронизации дерева с файлом (загрузка / reload), чтобы дальше проверять «файл на диске не менялся снаружи» перед записью.

---

## 2. Единственный редактируемый файл

- Путь от корня репозитория: `code_analysis/core/cst_tree/models.py`
- В этом шаге **не** создавать новые файлы и **не** менять `tree_builder.py`, `tree_saver.py`, команды.

---

## 3. Точка правки в коде

1. Открыть [`models.py`](../../../../code_analysis/core/cst_tree/models.py).
2. Найти объявление **`@dataclass` / `class CSTTree:`** и список полей до `loaded_at` / `last_accessed_at` (см. grep в репо: `class CSTTree`).
3. **Остановиться** перед строкой **`# cst-node-ids: begin`** — ниже не вносить изменений.

---

## 4. Поля для добавления (контракт)

Добавить на `CSTTree` минимум:

| Поле | Тип | Смысл |
|------|-----|--------|
| `disk_source_sha256_hex` | `Optional[str]` | SHA-256 **UTF-8 текста файла** в момент последней синхронизации с диском; `None` = «снимок не задан» (дерево из строки без файла и т.п.). |
| `disk_source_length` | `int` | Длина в байтах того же текста (для быстрой диагностики в логах); `0` если нет снимка. |

Опционально (если нужно для replay без повторного чтения — согласовать со [шагом 03](./03-code_analysis-core-cst_tree-tree_save_verification.md)):

- `disk_sync_generation: int` — монотонно увеличивать при каждом успешном reload/load из файла (не обязательно в шаге 01).

**Значения по умолчанию:** `disk_source_sha256_hex=None`, `disk_source_length=0`, чтобы существующие вызовы `CSTTree.create(...)` не ломались — `create` не обязан передавать новые поля, если у dataclass есть default.

---

## 5. Ограничения и риски

- **Не** редактировать блок между `# cst-node-ids: begin` и `# cst-node-ids: end`.
- **Не** делать `CSTTree` frozen, если он сейчас мутабельный — только добавить поля.
- Имена полей зафиксировать как в таблице выше — [шаг 02](./02-code_analysis-core-cst_tree-tree_builder.md) будет их заполнять; [шаг 03](./03-code_analysis-core-cst_tree-tree_save_verification.md) читать.

---

## 6. Критерии готовности (чеклист)

- [ ] Поля добавлены только в область `CSTTree`, до `# cst-node-ids: begin`.
- [ ] Все новые поля имеют значения по умолчанию; `CSTTree.create` компилируется без изменений или с минимальным явным присваиванием в `create`, если потребуется.
- [ ] `flake8` / `mypy` на `models.py` без новых ошибок.
- [ ] После этого шага репозиторий собирается; заполнение полей — в шаге 02.
