# Шаг 0: верификация `project_id` (без целевого файла в репозитории)

**Целевой артефакт:** подтверждённый UUID проекта в ответе **code-analysis-server**; в этом шаге **нет** одного файла кода в дереве репозитория — только проверка через MCP.

## Цель

Перед любыми правками через сервер (`cst_*`, `read_project_text_file`, `universal_file_replace`, …) убедиться, что **`project_id`** репозитория присутствует в **`list_projects`**.

## Действия

1. Вызвать **`list_projects`** на code-analysis-server (MCP proxy: `call_server`, `command: list_projects`, `params: {}`).
2. Сверить UUID из корневого **`projectid`** (или из списка проектов) с ответом сервера.
3. При отсутствии проекта — **остановиться**: зарегистрировать проект / исправить конфиг; **не** выполнять серверные правки с предполагаемым UUID.

## Критерий готовности

- Целевой **`project_id`** явно подтверждён ответом сервера; все последующие команды используют только его.
